# AI 代码分析智能体系统（基于 GitHub + DeepSeek API）
## 1. 产品定位
自动监听开发者 GitHub 代码提交，通过 **Git diff + DeepSeek AI** 自动分析：
- 真实工作量
- 代码改动详情
- 提交是否有效
- 是否摸鱼
- 开发者能力评级
**全程无需人工管理、无需开会、无需填工时。**

---

## 2. 核心功能（必做）
1. **GitHub WebHook 接收服务**
   - 接收 push 事件
   - 获取：提交人、时间、before、after、文件列表

2. **代码差异自动拉取**
   - 本地克隆仓库
   - 执行 `git fetch` + `git diff`
   - 解析：新增/删除/修改文件、行数

3. **DeepSeek AI 自动分析**
   - 输入：diff 代码
   - 输出：固定 JSON 结果
     - 工作量：低 / 中 / 高
     - 是否有效提交（是/否）
     - 代码质量简述
     - 开发者等级：初级/中级/高级
     - 是否疑似摸鱼（是/否）

4. **结果存储与展示**
   - 存储每次提交分析结果
   - 支持按人/按天/按仓库查看

5. **预警（可选）**
   - 连续无提交
   - 大量无效提交
   - 疑似摸鱼

---

## 3. 技术栈（Cursor 可直接识别）
- 后端：Python 3.9+
- Web 框架：Flask
- 代码仓库：GitHub
- AI 模型：DeepSeek API
- 存储：SQLite（简单版）
- 执行命令：git diff、git fetch

---

## 4. 系统流程（开发必读）
```
1. 开发者 git push
2. GitHub 发送 WebHook 到后端 /webhook/github
3. 后端解析：
   - pusher.name
   - before commit
   - after commit
4. 后端执行：
   git fetch
   git diff before after
5. 将 diff 内容传给 DeepSeek API
6. AI 返回结构化 JSON
7. 保存到数据库
8. 提供接口查询分析结果
```

## 5. API 设计（给 Cursor 直接开发）

### 5.1 接收 GitHub WebHook
- 路径：`POST /webhook/github`
- 参数：GitHub 推送的完整 JSON
- 返回：200 OK

功能：
- 验证请求合法（可选 secret）
- 提取提交人、before、after、时间
- 触发 diff 分析

---

### 5.2 代码差异分析服务（内部函数）
函数名：`analyze_commit(repo_path, before, after)`
返回：
```json
{
  "files": ["a.py", "b.js"],
  "insertions": 20,
  "deletions": 5,
  "total_lines": 25,
  "diff_content": "完整diff字符串"
}
```

---

### 5.3 DeepSeek AI 分析接口
请求地址：`https://api.deepseek.com/v1/chat/completions`
Headers：
```
Authorization: Bearer YOUR_DEEPSEEK_API_KEY
Content-Type: application/json
```

请求体（固定提示词，直接用）：
```json
{
  "model": "deepseek-chat",
  "temperature": 0.1,
  "messages": [
    {
      "role": "user",
      "content": "你是专业代码分析AI。下面是一段git diff代码，请你严格只输出JSON，不要任何多余文字。输出字段：workload（低/中/高）、is_valid_commit（布尔）、quality_comment（字符串）、level（初级/中级/高级）、is_suspected_slacking（布尔）。git diff：\n" + diff_content
    }
  ]
}
```

AI 返回格式（强制）：
```json
{
  "workload": "中",
  "is_valid_commit": true,
  "quality_comment": "业务逻辑正常，代码规范",
  "level": "中级",
  "is_suspected_slacking": false
}
```
