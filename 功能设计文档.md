## 1. 文档目的

本文档汇总前期关于「快原型」能力边界、与 Cursor / GitHub Copilot / Cline 等 IDE 辅助形态的对比、**用户本机文件系统 + Web 交互**、以及 **本地可执行程序（EXE）承接任务** 等讨论结论，形成可指导后续产品与技术拆分的**功能设计基线**。本文不替代接口规格说明（API Spec）与详细技术设计（TDD），但为二者划定范围与优先级。

---

## 2. 背景与问题陈述

### 2.1 当前 Web 端「快原型」能力（概要）

- **Spec 模式**：受约束的 JSON 结构 + 内置渲染器预览；导出为单份 `spec.json` 与单页 `preview/index.html`。
- **Mockup 模式**：生成单页 HTML/CSS 类产物，用于快速预览与下载。
- **局限**（已分析结论）：
  - **不生成**客户真实前端工程中的多文件源码（如 Vue SFC、路由、工程化资源）。
  - **不输出**多个独立「页面文件」或站点级路由；多屏语义主要通过 **Tabs / Panel** 等在**同一文档**内表达。
  - 数据流为：浏览器 → 业务后端 → 云端大模型（如 DeepSeek）→ 返回结构化结果或 HTML，**与用户本机工作区无直接绑定**。

### 2.2 目标体验（方向性描述）

用户希望在不牺牲安全的前提下：

- 将**文件存储与工作区**放在用户本机；
- Web 上的「快原型」作为**入口与轻量交互**，复杂编辑与上下文在**本地**完成；
- 能力上逐步接近「在 VS Code 类环境中，由 AI 辅助生成/修改前端界面」的体验，同时**尽可能在本地处理**敏感上下文与可选的本地大模型推理。

## 3. 设计目标与非目标

### 3.1 目标（Goals）

1. **本地工作区**：用户指定（或系统选择）的本机目录作为真源，支持多文件读写、监听、搜索。
2. **任务衔接**：用户从 SaaS「快原型」发起任务，能**可靠地**将任务上下文传递到本地程序并继续执行。
3. **可选云端模型**：在用户授权前提下使用 DeepSeek 等 API；密钥与传输策略需符合安全基线（见第 6 节）。
4. **可选本地模型**：支持接入用户本机 Ollama 等，减少代码与提示词出网。
5. **可实现的沙箱**：通过**限定语言、UI 组件库与依赖白名单**，提高代码生成可验证性与安装包可控性（见第 7 节）。

### 3.2 非目标（Non-Goals）— 首版可不承诺

1. 在纯浏览器环境（无本地组件）内，按用户**任意输入的绝对路径**静默读写磁盘。
2. 与 VS Code 插件生态完全兼容或复刻其全部扩展能力。
3. 无边界支持任意技术栈的 `npm install` 矩阵（除非单独定义「重量级沙箱」阶段）。

## 4. 总体架构方案

### 4.1 推荐形态：Web 入口 + 本地 EXE（或等价安装包）

| 层级 | 职责 |
|------|------|
| **Web（快原型）** | 项目选择、需求/《页面设计文档》编辑、发起「在本地继续」、鉴权与任务元数据托管。 |
| **本地程序** | 文件系统、多文件编辑器（如嵌入 Monaco）、文件监听、全局搜索、（可选）语义索引、调用本地或云端 LLM、在限定沙箱内执行构建/预览命令。 |
| **服务端** | 用户与租户鉴权、任务会话（短期 token）、可选的 **API 代理**（避免长期密钥落在客户端）。 |

### 4.2 Web 唤醒本地程序（可行性已确认）

浏览器无法在无用户许可下任意启动 EXE，需采用以下**标准机制之一**（可组合）：

1. **自定义 URL 协议**（安装时注册）：例如 `yourapp://prototype?session=<id>`，由本地程序拉取服务端任务详情（避免在 URL 中承载大 payload 与长期密钥）。
2. **本机常驻服务**：监听 `127.0.0.1:<端口>`，Web 通过 `fetch` 投递任务；须校验 **Origin / Token**，防止恶意站点滥用本机服务。
3. **降级**：用户手动打开本地程序并输入会话码（体验较弱，可作为 MVP 兜底）。

**推荐**：自定义协议 + **服务端短期 session**，任务体、文件列表、引用资源通过 HTTPS 由 EXE 凭用户登录态拉取。

### 4.3 纯浏览器路径（补充说明）

若坚持「仅安装浏览器、不装 EXE」，在 Chromium 系可使用 **File System Access API**（用户**选择文件夹**而非任意路径字符串），能力限于：选定目录树内的读写、无完整 IDE 级监听/终端/Git。与「指定绝对路径 + 大仓语义索引 + 类 VS Code」目标不一致时，**不应作为唯一方案**。

## 5. 与「IDE 内 AI 写真实源码」的关系

| 维度 | Cursor / Copilot / Cline（VS Code） | 当前 Web 快原型 | 本地 EXE 方案 |
|------|-------------------------------------|-----------------|---------------|
| 产物 | 真实仓库内源码文件 | Spec / 单页 HTML | **可定位为真实源码**（若在沙箱模板内） |
| 上下文 | 工作区索引、打开文件、诊断信息 | 单次 prompt 为主 | **可扩展**为目录索引 + 诊断回路 |
| 闭环 | 编辑 → 构建/报错 → 再迭代 | 预览为主 | **可支持**固定脚手架下的 dev/build |

结论：本地 EXE + 固定技术栈 + 工作区绑定，是向「类 Copilot 写界面」靠拢的**必要条件**；仅靠现有 Web Spec/HTML 路径**不等价**于该体验。

## 6. 安全与密钥（DeepSeek 等）

### 6.1 原则

- 传输层：**HTTPS** 为基线。
- **优先**不在 EXE 内长期保存明文 API Key；可采用：
  - **服务端代理**：浏览器/EXE 仅持用户会话，由服务端代调模型；
  - 或 **短期令牌**：服务端下发一次性/短寿命凭证，由 EXE 换取推理次数。
- 若必须本地存 Key：使用系统级密钥环（如 Windows DPAPI）加密落盘，并在隐私政策中说明风险边界。

### 6.2 本地服务防滥用

- 本机 HTTP 服务必须 **Token + 可选绑定本机用户**，限制仅本机回环访问，并记录敏感操作审计（本地日志可选上传需另议）。

## 7. 沙箱与固定技术栈（提高「实现度」的策略）

前期结论：**将语言、组件库与依赖限制为白名单**，可显著提高可落地性。

| 方面 | 收紧后的收益 |
|------|----------------|
| 安装包 | 可捆绑**单一版本**运行时与工具链，减少组合爆炸 |
| 校验 | 可对 `package.json` / import 做**允许列表**检查 |
| AI 生成 | 系统提示词与 RAG 文档只需覆盖**一套**组件 API |
| 构建与预览 | 固定脚手架，错误模式可预测，便于自动重试与提示 |

**产品代价**：仅支持声明过的技术栈（例如 Vue 3 + 指定 UI 库 + Vite）；异构项目需迁移或使用模板工程。该取舍与「提高实现度」目标一致，建议在文档与 onboarding 中明确公示。

## 8. 能力分阶段建议

### 阶段 A — MVP（验证闭环）

- 自定义协议唤醒 + session 拉取任务；
- 工作区目录选择、多文件编辑（Monaco 等）、保存；
- 全局文本搜索（如 ripgrep 捆绑或等价实现）；
- 云端模型（服务端代理或短效令牌）或用户粘贴 Key 的 DeepSeek 调用；
- **固定模板项目** + 白名单依赖，可选 `npm run dev` 级预览（若捆绑 Node）。

### 阶段 B — 增强

- 文件系统监听、增量索引；
- 本地 LLM（Ollama）一键接入与切换；
- 基础语义检索（向量库嵌入，限定仓库规模与忽略规则）。

### 阶段 C — 进阶（可选）

- 更深 Git 集成、终端、多包管理器策略评估；
- 与设计工具或截图输入的对接（独立立项）。

## 9. 与现有 AutoAttend 后端的衔接（概念）

- Web 端保留「创建快原型会话 / 导出《页面设计文档》」等能力；
- 新增或与现有鉴权打通：**`prototypeSessionId`**（示例）关联 prompt、报价项目引用、版本号；
- EXE 使用用户 Token + `prototypeSessionId` 拉取 payload，写入用户本地工作区；
- 可选：会话结束后由用户选择是否将产物摘要或文件列表回传云端（**默认不回传源码**，需合规与设置项）。

具体 REST/MQ 形态在接口设计中展开，本文仅锁定**数据所有权与流向**原则。

## 10. 风险与待决事项

| 项 | 说明 |
|----|------|
| 混合内容 | HTTPS 页面调用 `http://127.0.0.1` 需按浏览器策略验证与降级方案 |
| 安装与更新 | EXE 签名、自动更新通道、企业环境策略 |
| 许可证 | 捆绑的 Node / 工具链与 GPL 等许可证合规 |
| 大仓性能 | 索引与搜索的资源上限与 `.gitignore` 策略 |
| 多租户 | 本地数据默认属用户；云端仅存会话元数据时的留存周期 |
