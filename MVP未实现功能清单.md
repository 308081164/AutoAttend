# MVP 未实现功能清单（基于 docs 与代码对照）

> 说明：此清单基于 `docs/` 下功能设计文档中标注的「规划/后续迭代/不在范围」与当前前后端路由/接口实现做对照整理，用于后续排期与验收。

## A. 项目协作（多维表）

- [x] **词云图**：多维表仪表盘的“问题词云图”目前为预留占位（待实现分词、停用词、权重与渲染）。
- [ ] **平均解决时长**：多维表仪表盘的“问题平均解决时长计数图”目前为预留占位（需定义完成态、按历史/时间字段计算）。
- [ ] **多视图**：甘特图 / 日历视图（文档明确不在本期范围）。
- [ ] **富文本讨论与 @提醒**：目前讨论为纯文本，@提醒/表情/富文本未实现。
- [ ] **大文件分片/更强预览**：文档提到的大文件分片、更多在线预览能力未实现。
- [ ] **更细粒度 RBAC**：当前为项目级权限与角色，细粒度 RBAC（字段/操作级）未实现。

## B. 报价 → 需求 → 合同

- [x] **与多维表“需求记录一键带入”**（MVP 已落地）：报价项目需已关联协作项目（`link_table_id` = `biz_project.id`，创建仓库并同步后会自动回写）。在报价页可 **「从多维表带入需求」**：从 **「项目调整」** 表（`issue_tracking`）读取「问题描述」「归属模块」，映射为报价模块 **「从多维表导入」** 下的功能点；可配合 **「导入多维表格」** 为同一协作项目 **懒创建「待开发功能清单」** 表（`feature_backlog`）并跳转协作页。详见 `docs/需求-报价-合同半自动化产出-功能设计文档.md` 与 `docs/多维表格协作管理-功能设计文档.md`。
- [ ] **客户自助录入、多角色审批流**：文档标注为后续迭代，未实现。
- [ ] **多合同模板/法务审核流/多版本对比**：文档标注为后续迭代，未实现。

## C. 报价 → 创建仓库 → 需求同步（新链路）

> 设计文档：`docs/报价-创建仓库-需求同步-功能设计文档.md`

- [ ] **幂等与重试完善**：MVP 需要对“仓库已存在 / webhook 已存在 / 多维表重复导入”做更完整的幂等策略与可选分步重试接口。
