# 管理后台首页（控制台）设计说明

## 1. 目标

- 将原先「一行文字链接」的入口，改为**分区卡片 + 预览 + 快捷操作**，降低到达核心功能的层级。
- 视觉参考阿里云控制台：**浅渐变底、白卡片、圆角与轻阴影、区块标题与右上角次要链接**。
- 保留首页现有**仓库筛选、资源总览、图表、提交表、每日总结、Diff/AI 分析**等能力，避免功能回退；这些能力归入「开发与数据看板」延续在首屏枢纽下方。

## 2. 信息架构

| 区块 | 作用 | 数据与行为 |
|------|------|------------|
| **主体信息（公司）** | 归属感与合规展示 | **公司名称**优先取 `GET /admin/quote/party-b-profile` 中法人「法定名称」，否则自然人「姓名」；无则显示占位文案。 **登录账号**为 `localStorage.autoattend_username`，可复制。**企业认证**当前版本前端默认「已认证」，预留 `enterpriseVerified` 供后续对接。入口链至报价配置页维护乙方主体。 |
| **枢纽区布局** | 两排 × 每排三个 | 第一排：报价系统、API 配置与能力测试、团队管理；第二排：项目管理、**快捷运维**（占位）、**创意市集**（占位）。 |
| **报价系统** | AI 报价与工作台入口 | `GET /admin/quote/projects?page=1&pageSize=5` 展示近期项目列表与 `total`；快捷：新建报价、全部项目、报价参数配置。 |
| **API 配置与能力测试** | AI 与联调入口 | `GET /admin/ai-analysis/config`、`GET /admin/ai-analysis/qwen-config` 预览 Key 是否已配、服务是否开启；快捷：AI 配置页、能力测试页。 |
| **团队管理** | 协作账号 | `GET /admin/team/members` 统计成员数；快捷进入 `/team`。 |
| **项目管理** | 代码仓与协作 | 仓库下拉（与全页筛选联动）；未选仓时展示 `statsOverview` 三指标；快捷进入协作项目列表 `/collab/projects`。 |
| **快捷运维 / 创意市集** | 预留 | 仅占位文案与「敬请期待」，无路由。 |
| **开发与数据看板** | 原有看板 | 项目信息/资源总览、提交趋势、开发者排名、每日总结、最近提交等，逻辑与接口不变。 |

## 3. 前端实现要点

- 路由仍为 `/`，组件 **`DashboardView.vue`**。
- 主体信息：公司名来自报价 **乙方主体模板**（与 `/quote/config` 同源接口）；登录账号仍为 `autoattend_username`。
- 各枢纽区块失败时静默降级（列表空、统计为 `—`），不阻断整页。
- 国际化：文案键放在 `dashboard.console*`（`zh.js` / `en.js`）。

## 4. 验收关注

- 管理员登录后首屏可见六个枢纽卡片 + 主体信息；占位卡片无跳转；其余快捷入口路由正确。
- 更换「项目」下拉后，下方图表与提交列表筛选行为与改版前一致。
- 报价列表预览最多 5 条，与列表页数据结构一致（`id`、`name`、`techStack` 等）。
