## 2. 背景与改名动机

### 2.1 现状

当前平台已有以下能力模块：

| 模块 | 现状 |
|------|------|
| **报价系统** | 已实现需求录入、人天计算、风险调整、报价单生成、AI 合同生成、多维表联动等完整链路 |
| **机会角** | 已设计「人才角」与「项目角」，但尚未实现 |
| **咸鱼值守** | 待开发（见《咸鱼平台值守能力接入-功能设计文档》） |

### 2.2 改名动机

「报价系统」这个名称过于狭窄，无法涵盖以下场景：

1. **商务沟通**：与客户在淘宝/咸鱼上的聊天互动
2. **商机获取**：从项目角发现并申请项目机会
3. **报价与签约**：现有报价→合同链路
4. **客户管理**：潜在客户、意向客户、成交客户的分层管理

**新名称建议**：**「商务系统」**（Business System）或 **「商机工作台」**（Opportunity Workspace）

> 推荐使用 **「商务系统」**，简洁且能覆盖从商机获取到签约交付的全链路。

## 3. 商务系统整体架构

```
商务系统 (Business System)
├── 1. 商机管理 (Opportunity Management)
│   ├── 项目角（浏览/申请项目机会）
│   ├── 人才角（展示团队能力，吸引项目方）
│   └── 商机看板（汇总所有潜在机会）
├── 2. 客户沟通 (Customer Communication)
│   ├── 咸鱼值守（多账号消息监听与回复）
│   ├── 淘宝聊天（淘宝千牛/旺旺消息接入）
│   └── 沟通记录（跨平台消息归档）
├── 3. 报价与合同 (Quoting & Contracting)
│   ├── 解决方案报价（多交付物报价）
│   ├── 价格锚定与调整
│   ├── 报价单生成与导出
│   └── AI 合同生成
├── 4. 客户管理 (Customer Management)
│   ├── 客户档案（基本信息、沟通历史、报价历史）
│   ├── 客户分层（潜在/意向/成交/流失）
│   └── 跟进记录
└── 5. 数据看板 (Dashboard)
    ├── 商务漏斗（机会→报价→合同→成交）
    ├── 业绩统计
    └── 渠道分析（咸鱼/淘宝/自拓）
```

## 4. 模块一：商机管理

### 4.1 项目角（复用机会角设计）

从《机会角-功能设计文档》中的「项目角」迁移至商务系统，作为商机获取的核心入口。

| 功能 | 说明 |
|------|------|
| **项目列表** | 展示正在招募开发者的项目（背景、角色、收益、发起者） |
| **筛选与搜索** | 按技术栈、收益类型、远程/驻场、周期、领域筛选 |
| **项目详情** | 查看完整项目信息，包括发起人信誉、历史项目 |
| **申请参与** | 开发者申请参与项目，发起人审核通过/拒绝 |
| **收藏项目** | 收藏感兴趣的项目，后续跟进 |
| **我的申请** | 查看已申请项目的状态（待审核/已通过/已拒绝） |

### 4.2 人才角（复用机会角设计）

展示团队/个人开发者的技术能力，吸引项目方主动联系。

| 功能 | 说明 |
|------|------|
| **团队档案** | 展示团队名称、技术栈、项目经历、协作方式 |
| **案例展示** | 展示历史项目案例（可关联报价系统中的历史项目） |
| **协作评价** | 项目关闭后发起人对参与者的评价聚合展示 |
| **联系入口** | 项目方可通过站内信或咸鱼/淘宝联系 |

### 4.3 商机看板

统一视图，汇总所有潜在商机。

| 视图 | 说明 |
|------|------|
| **待处理商机** | 来自咸鱼/淘宝的新客户咨询、项目角的申请通知 |
| **跟进中商机** | 已建立联系、正在沟通报价的项目 |
| **近期成交** | 最近完成的签约项目 |
| **商机漏斗** | 从初次接触到签约的转化率统计 |

## 5. 模块二：客户沟通

### 5.1 咸鱼值守

详见《咸鱼平台值守能力接入-功能设计文档》，核心能力：

- 多账号同时登录咸鱼网页版
- 实时消息监听与提醒
- 快捷回复与文件发送
- 自动发货（虚拟商品）

### 5.2 淘宝聊天接入

#### 5.2.1 可行性分析

| 方式 | 可行性 | 说明 |
|------|--------|------|
| **千牛工作台 Web 版** | ✅ 可行 | 千牛有 Web 版（`https://work.taobao.com`），支持消息管理，可通过浏览器自动化接入 |
| **阿里旺旺 Web 版** | ✅ 可行 | 阿里旺旺有 Web 版，但功能有限 |
| **淘宝开放平台（TOP）** | ⚠️ 需申请 | 淘宝开放平台提供消息推送 API，但需要商家资质审核 |
| **千牛开放平台** | ⚠️ 需申请 | 千牛提供插件开发能力，但需要入驻开发者 |

#### 5.2.2 推荐方案

**Phase 1**：通过浏览器自动化（Puppeteer）接入千牛 Web 版，实现消息监听与回复（与咸鱼方案一致）。

**Phase 2**：若业务量增长，申请千牛开放平台或淘宝开放平台 API，获得更稳定、合规的接入方式。

#### 5.2.3 功能清单

| 功能 | 说明 |
|------|------|
| **千牛账号管理** | 添加/删除千牛账号，扫码登录 |
| **消息监听** | 监听买家咨询消息，实时推送提醒 |
| **快捷回复** | 预设回复模板，支持文字/图片/链接 |
| **订单查询** | 查看关联订单状态（待付款/已付款/已发货） |
| **消息归档** | 跨平台（咸鱼+淘宝）消息统一归档到客户档案 |

### 5.3 沟通记录统一管理

| 功能 | 说明 |
|------|------|
| **跨平台消息聚合** | 同一客户在咸鱼、淘宝的消息统一展示 |
| **客户画像** | 基于沟通记录自动提取客户需求、预算、决策链等信息 |
| **沟通标签** | 为沟通记录打标签（如「已报价」「意向强」「需跟进」） |
| **搜索** | 全文搜索历史沟通记录 |

## 6. 模块三：报价与合同（现有能力迁移）

将现有报价系统完整迁移至商务系统，保持功能不变，优化入口与交互。

### 6.1 迁移内容

| 现有功能 | 商务系统中的位置 |
|----------|-----------------|
| 报价项目创建/编辑 | 报价与合同 → 新建报价 |
| 需求结构化录入 | 报价与合同 → 需求录入 |
| AI 智能录入 | 报价与合同 → AI 辅助 |
| 人天计算与风险调整 | 报价与合同 → 报价计算 |
| 报价单生成 | 报价与合同 → 产出物 |
| AI 合同生成 | 报价与合同 → 合同管理 |
| 多维表联动 | 报价与合同 → 关联协作 |

### 6.2 新增优化

| 优化项 | 说明 |
|--------|------|
| **从沟通记录快速创建报价** | 在咸鱼/淘宝聊天界面一键「转为报价」，自动带入客户信息 |
| **报价模板** | 基于历史报价保存为模板，快速复用 |
| **报价版本对比** | 同一项目的不同报价版本对比 |
| **报价分享** | 生成报价单分享链接，客户在线查看 |

## 7. 模块四：客户管理

### 7.1 客户档案

| 字段 | 说明 |
|------|------|
| **基本信息** | 名称、联系方式、公司、行业 |
| **来源渠道** | 咸鱼/淘宝/项目角/自拓 |
| **沟通历史** | 跨平台消息记录汇总 |
| **报价历史** | 关联的报价项目与报价单 |
| **合同历史** | 关联的合同文档 |
| **成交记录** | 已签约项目与金额 |
| **标签** | 自定义标签（如「高意向」「老客户」「价格敏感」） |

### 7.2 客户分层

| 分层 | 定义 | 动作 |
|------|------|------|
| **潜在客户** | 首次咨询，尚未报价 | 发送产品介绍/案例，引导需求沟通 |
| **意向客户** | 已沟通需求，正在报价阶段 | 跟进报价进度，解答疑问 |
| **成交客户** | 已签约 | 交付跟进，售后服务 |
| **流失客户** | 超过 3 个月无互动 | 定期回访，发送新品通知 |

### 7.3 跟进管理

| 功能 | 说明 |
|------|------|
| **跟进记录** | 每次沟通后记录跟进内容与下一步计划 |
| **跟进提醒** | 设置定时提醒，避免遗漏重要客户 |
| **跟进看板** | 按阶段展示所有客户的跟进状态（看板视图） |

## 8. 模块五：数据看板

### 8.1 商务漏斗

```
曝光（项目浏览/咨询）→ 沟通（建立联系）→ 报价（发送报价单）→ 谈判 → 成交
```

每个阶段展示数量与转化率，帮助识别瓶颈环节。

### 8.2 业绩统计

| 指标 | 说明 |
|------|------|
| **本月新增商机** | 本月新增的潜在客户/项目申请数量 |
| **本月报价数** | 本月发送的报价单数量 |
| **本月成交额** | 本月签约合同总金额 |
| **成交率** | 报价→成交的转化率 |
| **平均成交周期** | 从首次接触到签约的平均天数 |
| **渠道分布** | 各渠道（咸鱼/淘宝/项目角）带来的商机与成交占比 |

### 8.3 团队协作

| 功能 | 说明 |
|------|------|
| **商务人员业绩看板** | 每个商务人员的商机、报价、成交数据 |
| **客户分配** | 管理员可将客户分配给不同的商务人员跟进 |
| **协作备注** | 多人可对同一客户添加备注，避免信息孤岛 |

## 9. 数据模型扩展

### 9.1 新增表

```sql
-- 客户表
CREATE TABLE aa_customer (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  tenant_id BIGINT NOT NULL,
  name VARCHAR(100),
  company VARCHAR(200),
  phone VARCHAR(50),
  email VARCHAR(200),
  source VARCHAR(50) COMMENT 'xianyu/taobao/project_corner/self',
  stage VARCHAR(20) DEFAULT 'lead' COMMENT 'lead/intent/quoted/negotiating/won/lost',
  tags JSON,
  assigned_to BIGINT COMMENT '负责员工ID',
  last_contacted_at DATETIME,
  created_at DATETIME,
  updated_at DATETIME
);

-- 跟进记录表
CREATE TABLE aa_customer_followup (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  customer_id BIGINT NOT NULL,
  content TEXT,
  next_action VARCHAR(500),
  next_action_at DATETIME,
  created_by BIGINT,
  created_at DATETIME
);

-- 商机表（从项目角申请产生的商机）
CREATE TABLE aa_opportunity (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  tenant_id BIGINT NOT NULL,
  source VARCHAR(50) COMMENT 'project_corner/xianyu/taobao',
  source_project_id BIGINT COMMENT '关联的项目角招募ID',
  customer_id BIGINT,
  status VARCHAR(20) DEFAULT 'new' COMMENT 'new/contacting/quoting/negotiating/won/lost',
  estimated_amount DECIMAL(12,2),
  created_at DATETIME,
  updated_at DATETIME
);
```

### 9.2 现有表关联

| 现有表 | 关联方式 |
|--------|----------|
| `biz_quote_project` | 增加 `customer_id` 字段关联客户表 |
| `biz_quote_document` | 通过报价项目关联客户 |
| `aa_xianyu_conversation` | 增加 `customer_id` 字段关联客户表 |
| `aa_xianyu_message` | 通过会话关联客户 |

## 12. 非功能需求

| 需求 | 说明 |
|------|------|
| **权限控制** | 商务系统数据按租户隔离，支持员工级权限（如仅查看自己负责的客户） |
| **数据安全** | 咸鱼/淘宝 Cookie 加密存储；客户联系方式脱敏展示 |
| **性能** | 咸鱼值守独立部署，不影响主应用性能 |
| **可扩展** | 预留「拼多多」「抖音电商」等平台接入接口 |
| **审计日志** | 记录所有商务操作（报价修改、合同生成、客户信息变更等） |
