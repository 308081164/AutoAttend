# 管理后台首页 — 数据与图表展示分析

## 一、当前首页现状

- **状态概览**：6 个数字卡片（活跃编码 / 评审中 / 审别人 / CI 修复 / 阻塞 / 空闲），目前仅「活跃编码」有真实数据（等于 commit 条数），其余为 0。
- **最近提交**：纯表格，列包括仓库、commit、作者、时间、说明、操作。
- **开发者**：纯表格（按仓库筛选时才有数据），列包括作者、邮箱、提交数、最近提交时间。
- **Diff + AI 分析**：点击「查看 Diff」后展开，以文字/表单为主。

整体缺少图形、趋势和汇总视图，信息密度高但不够直观。

---

## 二、系统当前能统计与获取的数据

| 数据来源 | 可用字段 / 能力 | 说明 |
|----------|-----------------|------|
| **Commit** | `repoFullName`, `commitSha`, `authorName`, `authorEmail`, `committedAt`, `message`, `filesChanged`, `insertions`, `deletions`, `validCommit`, `validReason` | 按仓库/全局分页列表；总条数 |
| **Dashboard** | `summary`（含 activeCoding 等）、`authors`（按仓库：authorName, authorEmail, commitCount, lastCommittedAt）、`commits` 列表 | 当前仅按单仓库或全量取 commit，无时间范围聚合 |
| **Repos** | 仓库名列表 | 来自 commit 表去重 |
| **AI 分析** | 单次 commit 的 workSummary, workType, isEffective, qualityLevel, qualityComment 等 | 按 commit 查询，暂无全局/聚合统计接口 |
| **协作（Collab）** | 项目列表、每项目下表格与记录数 | 需在管理端增加统计接口（如项目数、总记录数、最近更新） |

后端可扩展：按日/周 commit 数量、按仓库 commit 分布、按作者代码量、AI 分析结果分布（work_type / quality_level 统计）等。

## 三、软件行业通常关心的数据（与系统对应）

1. **提交与代码量趋势**  
   - 每日/每周 commit 数、新增/删除行数。  
   - **系统对应**：Commit 表有时间、insertions/deletions，可做时间序列聚合（需后端支持或前端用现有分页数据近似）。

2. **仓库与作者维度**  
   - 各仓库提交量占比、各作者提交量/代码量排名。  
   - **系统对应**：现有 `authors`（按仓库）、commit 列表可前端按 repo/author 聚合。

3. **单次提交质量（AI）**  
   - 工作类型分布（feature/bugfix/refactor 等）、质量等级分布、是否有效。  
   - **系统对应**：AI 分析结果表有 work_type、quality_level、is_effective，可新增聚合接口或在前端从已有接口汇总。

4. **协作与任务**  
   - 项目数、任务/记录总数、近期更新。  
   - **系统对应**：Collab 项目与记录表，可增加简单统计接口。

5. **资源总览**  
   - 仓库数、总 commit 数、总开发者数、AI 分析覆盖数等。  
   - **系统对应**：现有 count 与列表均可支撑。

## 四、首页适合展示的内容与推荐形式

### 4.1 顶部：资源与汇总卡片（保留并增强）

| 内容 | 形式 | 数据来源 |
|------|------|----------|
| 仓库数 | 数字卡片 | `/admin/repos` 的 items.length |
| 总提交数 | 数字卡片 | `/admin/commits` 的 total（或新接口 `/admin/stats`） |
| 开发者数 | 数字卡片 | 全量 commit 按 author 去重，或 dashboard 全仓库 authors 汇总 |
| 协作项目数 / 任务数 | 数字卡片 | 需后端：`/admin/collab/stats` 或等价 |

**形式**：一行 4～6 个卡片，大数字 + 短标签，可选小图标或色块区分。

---

### 4.2 提交趋势（折线图 / 柱状图）

- **横轴**：日期（近 7 天或 30 天）。  
- **纵轴**：当日 commit 数，或再加「新增行数 / 删除行数」双轴。  
- **形式**：折线图（趋势）或柱状图（每日量），可选按仓库筛选。  
- **数据**：  
  - 理想：后端新增 `GET /admin/stats/commits-by-day?range=7d&repoFullName=xxx` 返回 `[{date, count, insertions, deletions}]`。  
  - 或前端用现有 `/admin/commits` 多页/多 range 拉取后按日聚合（数据量不大时可行）。

---

### 4.3 各仓库提交占比（饼图 / 环形图）

- **内容**：各 repo 的 commit 数占比。  
- **形式**：饼图或环形图，点击可跳转或联动「最近提交」筛选。  
- **数据**：前端按 `/admin/commits` 的 items 按 `repoFullName` 分组计数；或后端 `GET /admin/stats/commits-by-repo` 返回 `[{repoFullName, count}]`。

---

### 4.4 开发者提交量排名（横向柱状图）

- **内容**：按 commit 数或代码量（insertions+deletions）排序的 Top N 作者。  
- **形式**：横向柱状图，一栏「提交数」或「代码变更行数」。  
- **数据**：  
  - 选「全部」时：需后端按全库 aggregate 作者（当前仅有按单 repo 的 authors），或前端用全量 commits 聚合。  
  - 选单仓库时：直接用 dashboard 的 `authors`。

---

### 4.5 状态概览（保留但改为更贴切的含义或图表）

- **现状**：除「活跃编码」外均为 0，含义偏「状态机」，与当前数据不完全匹配。  
- **建议**：  
  - **方案 A**：保留 6 宫格，但改为「近 7 天提交数」「有效提交数」「涉及仓库数」等可直接从现有数据算出的指标，仍用数字卡片。  
  - **方案 B**：改为「近 7 天每日提交数」小柱状图/折线，替代原 6 宫格，与 4.2 趋势可二选一或合并。

---

### 4.6 AI 分析概览（有条件时展示）

- **内容**：已有 AI 结果的 commit 中，work_type 分布（如 feature / bugfix / refactor）、quality_level 分布、effective 占比。  
- **形式**：小饼图或条形图，仅当「有 AI 结果」时显示该区块。  
- **数据**：需后端 `GET /admin/ai-analysis/stats` 返回聚合（如按 work_type、quality_level、is_effective 计数），或前端在已有列表中按 commit 逐个取 result 后汇总（成本高，不推荐）。

---

### 4.7 协作/项目摘要（可选）

- **内容**：项目数、总任务/记录数、最近更新项目。  
- **形式**：1～2 个数字 + 简短列表或「进入协作」入口。  
- **数据**：需后端提供 `/admin/collab/stats`（或等价），返回 projectCount、recordCount、lastUpdated 等。

---

### 4.8 最近提交（保留表格，增强信息）

- **保留**：表格形式，便于逐条操作（查看 Diff、运行 AI）。  
- **增强**：  
  - 可选展示「变更行数」（insertions/deletions）、filesChanged。  
  - 可选小图标区分「已有 AI 结果 / 无结果」。  
- **数据**：现有 `/admin/commits` 已含 stats 时直接展示；若当前未返回可让后端在 commit 列表里带上 insertions/deletions、filesChanged。

## 五、实现优先级与依赖建议

| 优先级 | 内容 | 形式 | 后端是否需要新增 |
|--------|------|------|-------------------|
| P0 | 资源汇总卡片（仓库数、总提交、开发者数） | 数字卡片 | 可选：统一 `/admin/stats` |
| P0 | 提交趋势（近 7 天） | 折线/柱状图 | 建议：按日聚合接口 |
| P1 | 各仓库提交占比 | 饼图/环形图 | 可前端聚合或新接口 |
| P1 | 开发者排名（按仓库） | 横向柱状图 | 现有 authors 即可 |
| P2 | 状态 6 宫格改为真实指标或小图 | 卡片/小图 | 否 |
| P2 | 最近提交表格增加列 | 表格 | 确认 commit 列表是否带 stats |
| P3 | AI 分析分布 | 饼图/条形图 | 是：AI 聚合接口 |
| P3 | 协作摘要 | 数字+入口 | 是：collab 统计接口 |

## 六、技术选型建议（前端）

- **图表库**：推荐 **Chart.js**（轻量、Vue 有封装）或 **ECharts**（能力更强、体积更大）。  
- **布局**：首页采用「上汇总卡片 → 中图表区（2～3 块）→ 下表格」的经典 Dashboard 布局，图表区用 grid 或 flex 多列。  
- **响应式**：图表随容器宽度自适应，移动端可考虑图表区单列或折叠。

---

## 七、小结

- **适合在首页展示的**：提交与代码量趋势、仓库/作者维度分布、资源与汇总数字、AI 分析分布（有数据时）、协作摘要（有接口时）。  
- **推荐形式**：数字卡片做总览，折线/柱状图做趋势与排名，饼图/环形图做占比，表格保留并增强用于「最近提交」与操作入口。  
- **最小可行**：先做「资源汇总卡片 + 提交趋势图 + 仓库占比图 + 开发者排名图」，用现有或少量新增接口即可让首页从纯文字/表格变为「图表 + 汇总 + 表格」的直观看板。
