# 前端「最近提交」不展示原因分析

## 现象

- 接口 `GET /api/admin/commits?page=1&pageSize=50` 返回 200，且 `data.items` 中有 27 条提交。
- 页面上仍显示「暂无提交记录」。

## 根因：两套接口、两种字段名

项目中存在**两个**返回提交列表的接口，且字段名不同：

| 接口 | 用途 | 返回中列表字段名 |
|------|------|------------------|
| `GET /api/admin/dashboard` | 首页概览（汇总、作者等） | **data.commits** |
| `GET /api/admin/commits`   | 分页提交列表（表格专用） | **data.items**   |

- 后端：`DashboardController` 使用 `data.put("commits", all)`，`CommitController` 使用 `data.put("items", pageItems)`。
- 前端「最近提交」表格**只**从 `loadCommits()` 取数，即只认 `/admin/commits` 的 **data.items**。

## 为什么「之前能展示、现在不能」的几种可能

1. **之前用的是 dashboard 的数据**
   - 若更早版本里「最近提交」来自 `loadDashboard()` 的 `resp.data.data.commits`，则当时展示的是 **data.commits**，不依赖 `/admin/commits`。
   - 后来改为用 `loadCommits()` 且只读 `resp.data.data.items` 后，若中间层（CDN/反向代理）改动了响应结构（例如把 `data` 展平），或 `code` 被转成字符串 `"0"`，就会导致 `resp.data.data` 为 `undefined` 或判断失败，从而 `this.commits` 一直为 `[]`，看起来「之前能展示现在不能」。

2. **响应被中间层改写**
   - 若 CDN/代理把 body 展平为 `{ code: 0, items: [...], total: 27, ... }`（没有 `data` 嵌套），则 `resp.data.data` 为 `undefined`，`resp.data.data.items` 取不到，列表为空。

3. **code 类型不一致**
   - 后端 `ApiResponse` 里 `code` 是 `int`，若某处改成字符串 `"0"`，则 `resp.data.code === 0` 为 false，不会进入赋值逻辑，列表也不会被设置。

## 已做修改

1. **以 dashboard 为主数据源**
   - `loadDashboard()` 成功且返回 `data.commits` 时**始终**用其设置 `this.commits`，不再仅在「当前列表为空」时兜底。
   - 首页加载时与 dashboard 同一次请求即可拿到提交列表，不依赖 `/admin/commits` 的解析是否成功。
   - dashboard 的响应解析兼容 `d.data || d`（展平结构）、`code === 0 || code === '0'`。

2. **loadCommits 解析更稳健**
   - 同时支持 `data.data.items` 与展平后的 `data.items`；成功条件为 `code === 0 || code === '0'`。
   - 仅当成功拿到数组时才覆盖 `this.commits`，失败时保留 dashboard 已设置的列表；「刷新」按钮仍走 loadCommits。

## 建议自检

- 在浏览器 Network 里看 `/api/admin/commits` 的响应 body：确认是 `{ code: 0, data: { items: [...] } }` 还是被展平。
- 若仍有问题，可临时在 `loadCommits` 里 `console.log(resp.data)`，确认实际结构后再对应调整取值路径。
