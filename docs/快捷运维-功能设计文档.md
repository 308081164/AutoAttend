# 快捷运维（多云资源统一监控与管理，内部代号 Nexus）— 功能设计文档

## 1. 文档说明

- **功能名称**：快捷运维 / 多云资源统一监控与管理平台（内部代号：**Nexus**）
- **版本**：v0.4（草案）
- **目标**：通过统一的 Web 界面，对公司在**阿里云**与**腾讯云**上的云服务器（ECS/CVM）资源进行集中化的**监控、管理与成本分析**，减少在多控制台间切换的成本，并为资源优化与成本控制提供数据支持。
- **扩展目标（v0.2）**：支持**同一云厂商下接入多个独立云账号**（含自有与代维客户），便于 MSP/外包运维场景；支持可配置的**自动巡检/同步间隔**；在系统内提供跳转**宝塔面板**、**唤起本地 SSH** 等快捷运维入口。
- **定位**：轻量级、安全、可扩展的**内部运维平台**（可覆盖「自有资源 + 代维多客户」；非面向公网 SaaS 的完整 CMP，除非后续单独立项扩展）。
- **依赖**：云厂商 OpenAPI/SDK、可长期运行的后端服务、关系型数据库、缓存与任务调度；与 AutoAttend 主系统是否共用账号与门户由集成方案决定（本文档不强制绑定）。

---

## 2. 功能目标与范围

### 2.1 目标

- **提升效率**：告别在多个云控制台之间切换，实现一站式运维入口。
- **统一视图**：跨云的资源健康度、性能指标与成本消耗聚合展示。
- **安全合规**：最小权限原则 + 操作审计，确保运维行为可追溯。
- **辅助决策**：为云资源优化、成本控制与架构调整提供数据依据。
- **代维效率**：多客户云账号分区管理、统一巡检节奏、一键跳转常用运维工具（面板 / SSH）。

### 2.2 范围

| 在范围内（按阶段见第 10 节） | 不在范围内（可后续迭代） |
|------------------------------|---------------------------|
| 阿里云 ECS、腾讯云 CVM 的元数据同步与列表/详情 | 非云主机类资源（对象存储、数据库托管等）的全量纳管（可二期） |
| **同一云厂商下配置多个独立云账号**（如多个阿里云主账号接入），并支持账户级展示名/标签（本公司、客户 A 等） | 云上子账号/RAM 的完整生命周期管理 UI（仅要求凭证可拉 API 即可） |
| 核心监控指标（CPU、内存、磁盘 I/O、网络等）的统一展示与告警 | 全链路 APM、日志检索一体化 |
| **用户可配置的自动巡检间隔**（与监控拉取、元数据同步、健康检查策略绑定，见 §6.6） | 巡检脚本在目标机内任意执行（需 Agent 时另立项） |
| **宝塔面板 URL 快捷跳转**、**本地 SSH 连接辅助**（命令复制或系统协议唤起，见 §5.5） | 系统内嵌 Web SSH 终端、无浏览器环境的一键登录 |
| 成本总览、构成分析与实例成本排行（基于账单 API 同步） | 多租户对外售卖、计费结算 |
| 凭证加密存储、RBAC、操作审计 | 与 AutoAttend 业务数据的自动关联（除非单独定义集成点） |
| 适配器模式统一多云数据模型 | 更多云厂商（华为云、AWS 等，见阶段三） |

### 2.3 术语补充

| 术语 | 含义 |
|------|------|
| **云账号接入** | 平台内一条可独立配置凭证与展示名的记录，对应一个云上的**独立账号维度**（可为贵司自有或某客户）；同一云厂商下可有多条接入。 |
| **自动巡检** | 见 §6.6：周期触发的元数据同步、监控拉取与（可选）告警评估的组合，**非**在客户机内执行自定义脚本的 Agent 巡检，除非另立项。 |

## 3. 用户角色与场景

| 角色 | 典型操作 |
|------|----------|
| **运维人员** | 查看实例列表与详情；执行启停、重启（在权限允许范围内）；配置告警与通知；处理告警；使用宝塔/SSH 快捷入口。 |
| **研发负责人** | 查看资源与成本大盘；识别高成本或闲置实例；参与容量与架构讨论。 |
| **平台管理员** | 维护多云账户与凭证、标签策略；分配 RBAC；审计日志查询与导出。 |
| **代维/服务商角色（可选）** | 为多个客户分别接入云账号；按客户筛选资源；配置巡检间隔；不跨客户泄露访问入口与凭证。 |

## 4. 功能模块总览

平台分为三大核心模块：**资源中心**、**监控中心**、**成本中心**；跨模块能力包括**多账号接入**、**自动巡检间隔**、**运维快捷入口**。

```
Nexus（快捷运维）
├── 资源中心：多云多账号接入、实例元数据、列表/详情、生命周期操作、宝塔/SSH 快捷入口
├── 监控中心：统一指标模型、仪表盘、告警规则与通知、可配置巡检/同步间隔
└── 成本中心：成本总览、构成分析、实例成本排行、账单同步
```

## 5. 资源中心

### 5.1 功能描述

作为平台基础，负责同步、展示和管理所有云服务器的**元数据**。

### 5.2 多云与多账号接入（含代维场景）

- **多账号、同厂商**：在**每个云厂商**（如阿里云、腾讯云）下，支持添加**多个独立云账号**接入，对应现实中「本公司主账号 + 若干客户主账号」等场景。每个接入在平台侧须有：
  - **展示名称**（如「本公司生产」「客户 A-阿里云」）；
  - **访问凭证**（AccessKey/SecretKey 或各云支持的等价接入方式）；**凭证须加密存储**（如 AES-256 或 KMS）；
  - **可选标签/分组**（环境、客户编码等），用于列表筛选与权限范围。
- **数据归属**：同步的 ECS/CVM 元数据、监控与成本均须带 **「云账号接入 ID」** 维度，避免多客户资源混淆。
- **说明**：云厂商侧的「主账号 / 子账号（RAM）」以各云定义为准；本平台强调的是**多个互不从属的云账号同时接入**，代维方使用**客户授权的子账号 AK** 亦可，只要 API 权限满足拉取实例与监控即可；**禁止**在文档与界面中鼓励使用主账号长期 AK，安全要求见 §9.1。

### 5.3 服务器列表总览

- 以列表形式展示所有已同步的 ECS 与 CVM 实例。
- **关键字段**：实例 ID、名称、**所属云账号接入**、云厂商、地域、IP 地址（公网/内网）、实例规格、当前状态（运行中/已停止等）、所属项目/标签。
- 支持按**云账号接入**、云厂商、地域、状态、名称等**筛选与搜索**。

### 5.4 实例详情与操作

- 点击实例进入详情页，展示更全面的配置信息。
- 提供安全操作：**启动、停止、重启**（具体以云 API 能力与账号权限为准）。
- **所有操作须写入审计日志**（操作人、时间、对象、结果）。

### 5.5 运维快捷入口：宝塔面板与本地 SSH

为减少在浏览器、云控制台、本地终端之间反复切换，在**实例列表或实例详情**提供快捷操作（具体入口以 UI 稿为准）：

| 能力 | 说明 | 约束与实现要点 |
|------|------|------------------|
| **跳转宝塔面板** | 用户可为实例配置**宝塔 Web 控制台地址**（完整 `https://host:port/...` 或「IP + 端口 + 路径」模板）；点击后在**新标签页**打开，由用户完成宝塔侧登录。 | 仅生成链接，平台不存储宝塔密码；若仅内网 IP，需用户本机网络可达（VPN/专线）。需 **HTTPS 证书风险提示** 等浏览器行为由用户自行确认。 |
| **本地 SSH 登录** | 提供**默认 SSH 命令**（如 `ssh root@公网IP`，用户名/端口可配置）；支持 **一键复制命令** 到剪贴板；可选支持 **`ssh://user@host:port`** 等格式唤起本机已安装的 SSH 客户端（依赖操作系统与浏览器是否注册协议处理器，**不保证所有环境可用**）。 | 平台不代管私钥；若使用密钥登录，可复制带 `-i` 的完整示例或由用户本地配置 `~/.ssh/config`。所有「复制/唤起」行为**记入审计**（不含密钥内容）。 |

- **可选字段**（实例级或接入级默认）：SSH 用户名、端口、宝塔 base URL 模板（便于批量新实例继承默认值）。

## 6. 监控中心

### 6.1 功能描述

聚合来自不同云厂商的监控数据，提供统一、直观的性能指标图表。

### 6.2 核心指标仪表盘

- 展示 **CPU 使用率、内存使用率、磁盘 I/O、网络流入/流出带宽**等核心指标的实时与历史趋势。
- 支持**自定义仪表盘**：用户选择关注的指标与实例（范围与配额由策略控制）。

### 6.3 统一数据模型

- 后端将阿里云与腾讯云不同格式的监控数据转换为**平台内部标准模型**再存储与展示，保证图表与告警规则的一致性。

### 6.4 智能告警

- 支持为关键实例配置告警规则（示例：CPU 使用率超过 80% 并持续 5 分钟）。
- 告警通知支持**邮件**、**Webhook**（如钉钉、企业微信）等渠道；渠道配置需安全存储（如 Webhook URL 脱敏展示）。

### 6.5 数据同步策略

- 采用后端**定时任务**异步拉取监控数据，避免前端高频调用云 API 带来的延迟与费用问题；需实现**限流与重试**，避免触发云厂商 API 配额限制。具体默认频率与 **§6.6 自动巡检间隔** 联动或为其上限约束。

### 6.6 自动巡检与时间间隔配置

「**自动巡检**」在本平台中指：按固定周期由后端触发的**例行任务集合**，至少包括（实现阶段可裁剪为 MVP 子集）：

- **元数据同步**：实例列表、状态、基础属性与云侧变更对齐；
- **监控指标拉取**：与 §6.2、§6.5 一致的核心指标入库；
- **（可选）健康/告警评估**：对已配置告警规则的实例执行条件判断并产生事件。

**间隔配置**（需在产品上可感知、可审计）：

| 配置项 | 说明 |
|--------|------|
| **全局默认间隔** | 平台级默认（如每 1 分钟 / 5 分钟 / 15 分钟 / 1 小时），作为未单独配置时的兜底。 |
| **按云账号接入覆盖** | 某一客户或某一接入凭证可单独设置更短或更长间隔（避免代维大量账号时 API 配额耗尽）。 |
| **（可选）按实例/实例组** | 核心实例高频、边缘实例低频，用于节省配额与存储。 |

**约束**：

- 最短间隔不得低于云厂商 API 与内部限流策略允许的**安全下限**（实现时写死或配置化）。
- 修改间隔须**记入审计**；间隔变更在**下一个调度周期**生效或立即重排（实现二选一并在界面说明）。
- 与 §6.5 的关系：若「巡检」与「监控拉取」合并为同一调度，则**一套间隔**驱动；若拆分，文档上需区分「同步周期」与「巡检评估周期」（可合并为同一配置项以降低用户心智负担）。

## 7. 成本中心

### 7.1 功能描述

提供跨云的**成本分析**与账单概览，辅助成本优化。

### 7.2 成本总览

- 展示近 7 天、近 30 天等周期的**预估/账单成本趋势**（以云账单 API 返回为准）。
- 支持按**云厂商、地域、实例类型、云账号接入**等进行成本构成分析（多客户代维时需按接入维度区分）。

### 7.3 实例成本排行

- 列出成本最高的 **Top N** 实例（如 Top 10），帮助识别资源浪费或异常。

### 7.4 账单数据同步

- 通过云厂商账单 API 定期同步账单明细（如阿里云 DescribeAccountBill、腾讯云 DescribeBillList 等）；同步周期与历史保留策略在实现阶段确定。

## 8. 技术实现要点（概要）

> 具体选型可在独立《技术方案》中定稿；此处仅保留与功能相关的架构约束。

| 层级 | 建议方向（可选） |
|------|------------------|
| 后端 | Python（Flask/FastAPI）或 Go（Gin）；接入 `aliyun-python-sdk`、`tencentcloud-sdk` 等 |
| 数据存储 | PostgreSQL 或 MySQL：用户、实例元数据、指标时序或聚合表、审计日志 |
| 缓存 | Redis：热点数据、会话（若与门户集成） |
| 定时任务 | Celery、XXL-JOB、或系统 Cron + 自建调度 |
| 前端 | React 或 Vue；图表 ECharts / Chart.js；组件库 Ant Design / Element 等 |

### 8.1 关键架构设计

- **适配器模式**：为阿里云、腾讯云分别实现适配器，将 API 响应转为平台标准模型。
- **队列与限流**：调用云 API 时通过消息队列或带限流的任务队列削峰，避免触发频率限制与雪崩。

## 9. 安全与合规性设计

### 9.1 凭证安全

- **最小权限**：在阿里云 RAM、腾讯云 CAM 中为平台创建**专用子账号**，仅授予只读与必要的控制类 Action（如 RebootInstance）；**禁止**在界面与培训材料中鼓励使用**主账号**长期 AccessKey；若业务上必须录入某客户主账号凭证，须通过**独立审批与加密强度**约束，并限定可见角色。
- **加密存储**：每个云账号接入的 AccessKey/SecretKey **分别**加密落库或对接 KMS；**禁止**明文落库与日志打印。
- **多客户隔离（代维）**：不同云账号接入之间的数据（实例、成本、审计、快捷入口配置）在权限模型上**默认互不可见**，除非平台管理员显式授权跨客户视图（可选能力）。

### 9.2 操作审计

- **平台侧**：记录登录、实例查看、启停/重启、告警规则变更、**自动巡检间隔变更**、**宝塔/SSH 快捷操作（打开链接、复制命令、协议唤起）**等关键操作（操作人、时间、对象、结果）。
- **云侧**：确保通过平台发起的 API 调用可在 ActionTrail / CloudAudit 等侧追溯。

### 9.3 访问控制

- 平台用户登录；基于 **RBAC** 区分只读、操作、管理等角色（细则在权限矩阵中展开）。

## 10. 里程碑与开发路线

| 阶段 | 内容 |
|------|------|
| **阶段一：MVP** | **单厂商多账号接入**（或先单账号后快速扩多账号）；服务器列表同步与展示（含**云账号接入**维度）；核心监控指标（如 CPU、内存）图表；**全局默认自动巡检/同步间隔**；基础登录与操作审计。 |
| **阶段二：增强** | 实例启停、重启等管理能力；灵活告警规则与通知；**按云账号接入覆盖巡检间隔**；**宝塔面板跳转 + SSH 命令复制/唤起**；成本中心（概览与排行，支持按接入维度）。 |
| **阶段三：优化与扩展** | 同步性能与更多监控指标；更多云厂商；成本优化建议等智能化能力；可选实例级巡检频率、跨客户授权视图。 |

### 13.2 选型建议（非强制）

- **优先高契合、中低复杂度**：A3（磁盘快照）、A7（安全组只读）、A8（EIP 只读或受限写）、A12/A13（云监控分组/事件）、A16（ActionTrail）、A17（标签与资源组）。
- **能力最强但治理成本最高**：A1（云助手）、A2（OOS），建议配套**命令白名单、审批、全量审计**后再上线。
- **易与当前「不在范围内」条款冲突**：A14/A15（日志/APM 一体化），建议单独立项或在 §2.2 中显式修订范围后再接入。
