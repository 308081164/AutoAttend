## 2. 产品决策（已定稿 / 当前默认）

| 主题 | 决策 |
|------|------|
| **租户标识** | 使用 **slug**（小写字母、数字、连字符，2～64 位），见 `AdminAuthService` 校验逻辑；**不使用自定义域名**（与业务约定一致）。 |
| **域名策略** | **单域名 + 路径**；不强制多子域；后续若启用子域需单独立项（Cookie、CORS、Webhook 回调）。 |
| **主账号** | **手机号（E.164）+ 密码**；短信验证后续版本再接（见主报告附录）。 |
| **计费模型** | **未接支付**；免费额度与付费档位留待阶段 2，当前不阻塞隔离骨架。 |

## 4. HTTP API 清单（按前缀，阶段 1 起须带租户上下文）

> **约定**：管理端请求在鉴权通过后由 `AdminAuthFilter` 写入 `TenantContext`；协作端在 `CollabAuthFilter` 通过后根据 `biz_user.tenant_id` 写入；**未鉴权**接口（如 Webhook）通过请求头 **`X-Tenant-Id`**（可选，缺省为 `1`）指定租户。

| 前缀 | 用途 | 租户来源 |
|------|------|----------|
| `/api/admin/**` | 管理后台（仪表盘、提交、报价、AI、团队等） | 登录会话中的 `tenant_id` |
| `/api/collab/**` | 协作 API | 当前用户所属 `tenant_id` |
| `/api/webhooks/**` | GitHub Webhook | `X-Tenant-Id`（可选，默认 `1`） |
| `/api/platform/**` | 监测台 | 与租户业务隔离，不写入业务 `TenantContext` |

**说明**：具体 Controller 路径以代码为准；新增模块应复用上述租户策略。
