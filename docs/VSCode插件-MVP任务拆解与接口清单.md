## 3. 数据库与接口变更清单

## 3.1 数据库变更（MVP）

### A. 尽量不新增表（推荐）

MVP 首选复用现有协作表，不新增数据表，仅补充以下能力：

1. 记录更新接口支持版本校验（`updated_at` 或 `version`）；
2. 审计日志复用既有机制，若无则增加通用操作日志；
3. 传送门数据复用现有 `biz_project_portal_link`（已新增）。

### B. 可选新增表（仅当需要更强冲突控制）

`biz_collab_record_revision`（可选）：

| 字段 | 类型 | 说明 |
|---|---|---|
| id | BIGINT PK | 主键 |
| record_id | BIGINT | 记录 ID |
| version | BIGINT | 版本号，自增 |
| updated_by | BIGINT | 更新人 |
| updated_at | DATETIME | 更新时间 |
| patch_json | JSON/TEXT | 字段变更摘要 |

> 说明：若现有记录表已有 `updated_at` 并可满足并发检测，则不必新增该表。

## 3.2 接口清单（插件使用）

### 3.2.1 鉴权

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/auth/login`（复用） | 登录换取 token |
| POST | `/api/auth/refresh`（如有） | 刷新 token |
| POST | `/api/auth/logout` | 退出登录 |

### 3.2.2 项目与表格

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/collab/projects` | 我的项目列表 |
| GET | `/api/collab/projects/{id}` | 项目详情 |
| GET | `/api/collab/projects/{id}/table`（或等价） | 当前表结构/列定义 |
| GET | `/api/collab/projects/{id}/records?page=1&pageSize=50` | 记录分页 |

### 3.2.3 记录编辑

| 方法 | 路径 | 说明 |
|---|---|---|
| PUT | `/api/collab/records/{recordId}` | 更新记录（建议支持版本字段） |
| POST | `/api/collab/records` | 新建记录（可选，MVP 也可先不做） |

**更新请求体建议（示例）**

```json
{
  "fields": {
    "c1": "任务标题",
    "c2": "进行中"
  },
  "ifUnmodifiedSince": "2026-04-09T10:30:00Z"
}
```

**冲突响应建议**

- HTTP `409`
- body:

```json
{
  "code": 40901,
  "message": "记录已被其他用户更新，请刷新后重试",
  "serverUpdatedAt": "2026-04-09T10:31:11Z"
}
```

### 3.2.4 看板与统计

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/collab/projects/{id}/stats/overview`（示例） | 看板摘要 |
| GET | `/api/collab/projects/{id}/stats/trends`（示例） | 趋势数据 |

### 3.2.5 附件与传送门

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/collab/projects/{id}/attachments` | 上传附件 |
| GET | `/api/collab/attachments/{attachmentId}/download` | 下载附件 |
| GET | `/api/collab/projects/{id}/portal-links` | 传送门列表（只读） |

### 3.2.6 聚合接口（建议新增，优化首屏）

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/plugin/collab/projects/{id}/bootstrap` | 一次返回项目信息 + 默认表结构 + 看板摘要 + 传送门 |

**返回体建议（示例）**

```json
{
  "project": { "id": 3, "name": "AutoAttend" },
  "defaultTable": { "columns": [] },
  "dashboard": { "todo": 12, "doing": 5, "done": 20 },
  "portalLinks": [{ "id": 1, "label": "测试环境", "url": "https://test.example.com" }]
}
```

## 4. 插件 UI 原型结构稿

## 4.1 信息架构（IA）

```text
AutoAttend（Activity Bar 图标）
├─ 登录页（未登录）
└─ 工作台（已登录）
   ├─ 项目选择（下拉/搜索）
   ├─ 导航 Tab
   │  ├─ 多维表格
   │  ├─ 项目看板
   │  └─ 传送门
   └─ 主内容区
      ├─ 表格视图（筛选栏 + 表格 + 详情抽屉）
      ├─ 看板视图（指标卡 + 趋势图）
      └─ 传送门视图（链接胶囊列表）
```

### 4.2 关键页面草图（文字版）

### A. 登录页

- 输入：服务地址、账号、密码
- 操作：登录
- 状态：登录中、失败提示、证书/网络异常提示

### B. 多维表格页

- 顶部：项目名、表名、刷新、筛选、搜索
- 中部：数据表格（分页）
- 右侧抽屉：字段详情、评论、附件
- 底部状态条：最后同步时间、请求耗时、错误提示

### C. 项目看板页

- 顶部：时间范围切换（7d/30d）
- 中部：任务状态统计卡片
- 下方：趋势图（新增/完成）

### D. 传送门页

- 以飞书风格“链接胶囊”展示；
- 每个胶囊：图标 + 文本 + hover 显示完整 URL；
- 点击：外部浏览器打开。

## 4.3 关键交互流程

### 流程 1：登录并进入表格

1. 用户点击 Activity Bar 图标；
2. 填写登录信息；
3. 获取 token 成功后加载项目列表；
4. 进入默认项目的默认表格。

### 流程 2：编辑记录并保存

1. 用户编辑单元格；
2. 本地 UI 先乐观显示新值；
3. 请求保存成功：状态提示“已保存”；
4. 请求冲突：提示“远端已变更”，提供“刷新并重试”。

### 流程 3：查看看板并跳转传送门

1. 切到“项目看板”查看指标；
2. 切到“传送门”点击“测试环境”；
3. 默认系统浏览器打开目标 URL。
