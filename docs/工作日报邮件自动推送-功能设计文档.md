# 工作日报邮件自动推送 — 功能设计文档

## 1. 文档说明

- **功能名称**：工作日报邮件自动推送
- **版本**：v0.1
- **目标**：基于现有 Git 提交与活动数据，向员工自动发送工作日报邮件；支持管理员配置开关、推送周期、企业信息与当日附加信息，支持员工退订与排除名单，便于后续迭代开发。
- **依赖**：现有 AutoAttend 后端、GitHub Webhook 落库的 commit 数据（含 `author_email`）、管理员后台。

---

## 2. 功能目标与范围

### 2.1 目标

- 在**不购买飞书企业版**的前提下，利用**已有 commit 作者邮箱**与活动数据，实现「工作日报邮件」的自动推送。
- 管理员可**手动开关**、配置**推送时间与周期**、设置**企业名称与当日附加信息**，并可**排除部分员工**；员工可**主动退订**。
- 日报内容基于当日（或近 24 小时）该员工的提交与活动汇总，便于团队同步进展。

### 2.2 范围（本期）

| 在范围内 | 不在范围内（可后续迭代） |
|----------|---------------------------|
| 总开关、推送周期配置 | 回复「TD」解析退订（仅支持退订链接） |
| 基于 cron 的定时推送（每日/每周/每月等） | 多语言、多模板 |
| 从 commit 作者去重得到收件人，支持黑名单 | 邮件打开/点击统计 |
| 退订链接 + 退订状态落库 | 自定义 HTML 模板编辑 |

---

## 3. 用户角色与场景

| 角色 | 操作 |
|------|------|
| **管理员** | 开启/关闭日报推送；配置推送时间与周期；设置企业名称、当日附加信息；维护不推送名单；查看退订列表。 |
| **员工（收件人）** | 收到日报邮件；点击邮件内「退订链接」完成退订。 |
| **系统** | 按配置在指定时间生成日报、调用发信服务发送邮件。 |

---

## 4. 功能需求详述

### 4.1 总开关

- 配置项：**是否启用工作日报邮件推送**（布尔）。
- 仅管理员可修改。
- 关闭后，定时任务仍可运行，但**不执行发信**（可视为“暂停推送”）。

### 4.2 推送时间与周期（自动推送策略）

支持以下策略（可配置其一）：

| 策略 | 说明 | 示例 |
|------|------|------|
| 每个工作日 指定时刻 | 周一至周五，每天一次 | 每个工作日 10:00 |
| 每周 某几天 指定时刻 | 固定星期几 | 每周一、三、五 10:00 |
| 每周 某一天 指定时刻 | 仅一天 | 每周五 18:00 |
| 每天 指定时刻 | 每天一次 | 每天 09:00 |
| 每月 某日 指定时刻 | 每月固定日期 | 每月 1 日 10:00 |

- 实现建议：存 **cron 表达式** 或 **策略类型 + 参数**（如 `type=WORKDAY`, `time=10:00`；`type=WEEKLY`, `days=[1,3,5]`, `time=10:00`），由定时任务解析并在对应时间触发「生成 + 发送」流程。

### 4.3 收件人范围与排除

- **默认收件人**：从 `aa_commit` 表按 `author_email` 去重，得到「有提交记录的邮箱」列表；可选：与「员工表」合并（若存在）。
- **排除名单**：管理员可配置「不推送的邮箱」列表（黑名单）；这些邮箱**永不发送**，不受总开关影响（仅“不发给这些人”）。
- **退订**：员工点击退订链接后，该邮箱标记为「已退订」；已退订的邮箱**不再发送**，直到管理员将其从退订列表移除（若提供“重新订阅”功能）。

### 4.4 退订机制

- **退订链接**（本期实现）：  
  每封日报邮件底部包含「退订」链接，例如：  
  `https://<系统域名>/api/public/report/unsubscribe?token=<token>`  
  - `token` 与邮箱一一对应（可加密或随机 ID 映射邮箱），有效期内点击即将该邮箱标记为退订。
- **退订后**：该邮箱不再收到任何日报，直至管理员清除退订状态（或提供“重新订阅”链接）。
- **回复「TD」退订**：本期不实现；若后续实现，需接入收信（IMAP/邮件 webhook）并解析回复内容。

### 4.5 日报内容与模板

- **数据来源**：当日（或近 24 小时）该员工在 `aa_commit` 中的记录，按仓库、提交次数、最后提交时间等聚合。
- **可配置文案**：  
  - **企业名称**：用于邮件标题或正文抬头（如「XX 公司工作日报」）。  
  - **当日附加信息**：一段文本，可由管理员每日或按需修改，插入到邮件正文中（如会议提醒、注意事项等）。
- **邮件要素建议**：  
  - 标题：含企业名称、日期、如「【XX公司】工作日报 - YYYY-MM-DD」。  
  - 正文：企业名称、当日附加信息、该员工当日提交汇总（仓库、次数、最近一条 commit 信息等）、退订链接。

### 4.6 邮件发送方式

- 支持通过 **SMTP**（Spring Mail）或 **第三方邮件 API**（如阿里云邮件推送、SendGrid 等）发送。
- 发件人、SMTP/API 配置由管理员在后台或配置文件中维护，不硬编码。

## 5. 数据模型建议

### 5.1 日报推送配置表 `aa_report_config`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| enabled | TINYINT(1) | 总开关：1=启用，0=关闭 |
| schedule_type | VARCHAR(32) | 策略类型：WORKDAY / WEEKLY / DAILY / MONTHLY |
| schedule_param | VARCHAR(255) | JSON 或键值，如 {"time":"10:00","days":[1,3,5]} |
| cron_expression | VARCHAR(64) | 可选：直接存 cron，与 schedule_type 二选一 |
| company_name | VARCHAR(128) | 企业名称 |
| daily_extra_message | TEXT | 当日附加信息（可每日更新） |
| mail_from_name | VARCHAR(64) | 发件人显示名 |
| mail_from_address | VARCHAR(128) | 发件人邮箱 |
| smtp_host / api_key 等 | VARCHAR | 发信相关配置（或另表/加密存储） |
| created_at, updated_at | DATETIME | 创建/更新时间 |

### 5.2 收件人与退订表 `aa_report_recipient`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| email | VARCHAR(255) UNIQUE | 邮箱（与 commit author_email 对应） |
| display_name | VARCHAR(128) | 显示名（可选） |
| unsubscribed_at | DATETIME NULL | 退订时间，NULL 表示未退订 |
| unsubscribe_token | VARCHAR(64) | 退订链接用 token |
| created_at, updated_at | DATETIME | 创建/更新时间 |

### 5.3 不推送黑名单表 `aa_report_blacklist`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| email | VARCHAR(255) UNIQUE | 不接收日报的邮箱 |
| created_at | DATETIME | 加入时间 |

### 5.4 发送记录表 `aa_report_send_log`（可选，便于排查与统计）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| email | VARCHAR(255) | 收件人 |
| report_date | DATE | 日报日期 |
| sent_at | DATETIME | 发送时间 |
| status | VARCHAR(16) | success / failed |
| error_message | TEXT NULL | 失败原因 |

## 6. 接口设计建议

### 6.1 管理员端（需鉴权）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/admin/report/config | 获取日报推送配置 |
| PUT | /api/admin/report/config | 更新配置（开关、周期、企业名称、当日附加信息、发信配置） |
| GET | /api/admin/report/recipients | 收件人列表（含退订状态）；支持从 commit 作者同步/刷新 |
| GET | /api/admin/report/blacklist | 不推送名单列表 |
| POST | /api/admin/report/blacklist | 添加黑名单邮箱 |
| DELETE | /api/admin/report/blacklist/{id} | 移除黑名单 |
| POST | /api/admin/report/recipients/{id}/resubscribe | 取消退订（重新订阅） |
| POST | /api/admin/report/send-now | 可选：手动触发一次“立即发送”测试 |

### 6.2 对外（无需登录）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/public/report/unsubscribe | 退订链接落地页；参数 token=xxx，校验后标记该邮箱退订并返回结果页 |

## 7. 定时任务

- **执行器**：Spring `@Scheduled` 或 Quartz，按配置的 cron 或策略**每到一个触发点**执行一次。
- **流程**：  
  1. 若总开关关闭，直接结束。  
  2. 确定「本次应发送的日期」（一般为当天或前一个工作日）。  
  3. 从 `aa_commit` 聚合当日（或近 24h）按 `author_email` 的提交数据。  
  4. 收件人 = 有提交记录的邮箱 − 黑名单 − 已退订。  
  5. 读取配置中的企业名称、当日附加信息、发信配置。  
  6. 按人生成日报内容，调用发信接口发送；可选写入 `aa_report_send_log`。

---

## 8. 安全与隐私

- 发信配置（SMTP 密码、API Key）建议加密存储或使用环境变量。  
- 退订 token 建议不可预测（随机 + 与邮箱绑定），并设置有效期（可选）。  
- 管理员接口需鉴权（与现有管理员权限一致）。  
- 日报内容仅包含该员工自己的提交汇总，不包含他人数据。

---

## 9. 与现有系统关系

- **数据**：只读 `aa_commit`（及现有活动相关表），不修改现有 Git/Webhook 逻辑。  
- **身份**：收件人邮箱与 commit 的 `author_email` 一致；可与现有「员工/身份」表扩展关联（可选）。  
- **配置与任务**：新增配置表与定时任务，不影响现有 Webhook、看板、Dashboard 接口。

---

## 10. 后续迭代可选

- 支持「回复 TD 退订」：接入邮件接收（IMAP 或 邮件 webhook），解析发件人 + 正文关键词。  
- 多模板、多语言。  
- 邮件打开/点击统计。  
- 管理员自定义 HTML 模板（富文本或占位符模板）。  
- 按部门/项目过滤收件人或日报内容。
