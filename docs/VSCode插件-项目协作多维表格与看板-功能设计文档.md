# VSCode 插件：项目协作多维表格与看板 — 功能设计文档

## 1. 文档说明

- **功能名称**：AutoAttend VSCode 插件（多维表格/项目看板）
- **版本**：v0.1（MVP 设计）
- **目标**：让开发者在 VSCode 内直接查看与编辑项目协作多维表格、查看项目看板与关键统计，降低在飞书/钉钉/网页来回切换成本。
- **适用对象**：研发、测试、项目经理（重点面向研发日常高频使用）。

---

## 2. 背景与问题

当前开发者需要频繁切换到办公软件或 Web 页面查看/编辑多维表格，典型痛点：

1. 上下文切换频繁，打断编码流；
2. 多维表格更新滞后（需要主动切换页面刷新）；
3. 项目看板、任务进度、附件等信息在开发环境内不可直达；
4. 对研发而言，工具链不统一，协作效率偏低。

**设计目标**：将“查看 + 编辑 + 跟进”入口前置到 VSCode，做到“代码与协作同屏”。

---

## 3. 产品目标与范围

### 3.1 目标

- 在 VSCode 内提供项目选择、多维表格浏览/编辑、项目看板查看能力；
- 支持登录鉴权、租户隔离、项目权限控制；
- 支持常见字段编辑（文本、状态、人员、时间、附件基础操作）；
- 支持“开发与数据看板”核心指标查看；
- 提供稳定的同步与冲突处理策略。

### 3.2 范围（MVP）

| 在范围内 | 不在范围内（后续迭代） |
|---|---|
| 账号登录、Token 持久化、自动续期 | SSO（企业微信/钉钉/飞书）深度打通 |
| 项目列表、表格列表、记录分页查询 | 离线编辑后批量回放 |
| 单条记录编辑、状态快捷更新 | 复杂公式字段编辑器 |
| 附件查看链接与上传（基础） | 本地大文件断点续传 |
| 看板核心指标只读展示 | 自定义图表拖拽编辑 |
| 传送门链接查看/打开（可选） | 插件内完整项目管理后台 |

---

## 4. 用户场景

1. **开发者日常更新任务**
   - 打开 VSCode 侧边栏插件；
   - 选择当前项目与协作表；
   - 快速修改任务状态、补充备注、上传截图附件；
   - 无需切出 IDE。

2. **研发查看项目健康度**
   - 在插件内进入“开发与数据看板”；
   - 查看待办/进行中/已完成分布、最近提交/活跃趋势；
   - 快速定位当天风险任务。

3. **测试/联调快速跳转**
   - 通过插件展示的项目“传送门”链接，一键打开测试环境、日志平台、接口文档。

## 5. 总体架构

## 5.1 架构分层

- **VSCode Extension Host**
  - 命令注册、状态管理、配置存储、鉴权管理；
- **Webview UI**
  - 表格/看板界面（建议 Vue 或 React）；
- **AutoAttend Backend API**
  - 复用现有协作项目 API，补齐插件专用聚合接口；
- **DB**
  - 复用现有协作表、记录、附件、看板统计表。

### 5.2 通信模式

- 插件与后端通过 HTTPS REST 通信；
- 插件内 `Extension <-> Webview` 通过 `postMessage`；
- 关键更新采用“拉模式 + 局部轮询”：
  - 表格页：每 20~30 秒轻量刷新；
  - 用户主动编辑后立即本地乐观更新，再以服务端回包校正。

## 6. 功能设计

### 6.1 登录与鉴权

- 支持账号密码登录（MVP），返回访问 Token（及刷新 Token，如果后端支持）；
- Token 使用 VSCode SecretStorage 保存，不落明文；
- 鉴权失败自动跳转登录并给出明确提示。

### 6.2 项目与表格导航

- 左侧树结构：
  - 我的项目
  - 项目下多维表格
  - 项目看板
- 支持最近访问项目缓存；
- 支持按项目名/表名搜索。

### 6.3 多维表格查看

- 支持列定义渲染、记录分页、关键字段快速筛选；
- 支持点击行打开详情抽屉（字段、评论、附件）；
- 保持与 Web 端字段语义一致（字段类型、状态颜色、附件展示）。

### 6.4 多维表格编辑

- 单元格内联编辑（文本、数值、日期、状态）；
- 行级保存（自动保存 + 手动保存可配置）；
- 冲突策略（MVP）：
  - 提交时携带 `updatedAt/version`；
  - 若后端检测冲突，提示“远端已更新”，用户可选择覆盖或拉取最新后重试。

### 6.5 附件能力

- 支持在记录详情中上传附件（图片/文档/视频，复用现有后端能力）；
- 支持附件预览链接与下载跳转；
- 对大文件给出大小限制提示（MVP 不做断点续传）。

### 6.6 项目看板（只读）

- 显示项目核心统计卡片与趋势图（复用现有 Dashboard 接口）；
- 支持时间范围切换（近 7 天/30 天）；
- 不在插件内提供复杂图表配置，仅保证“可读 + 快速”。

### 6.7 传送门链接（增强项，建议纳入 MVP）

- 在插件项目首页展示当前项目传送门（外部网页、工具、测试地址）；
- 点击后使用 VSCode 外部浏览器打开；
- 若后端已支持项目级传送门 CRUD，可先做只读展示，编辑入口后续补齐。

## 7. 后端接口设计建议

> 原则：优先复用已有 `/api/collab/...` 能力；插件只新增必要聚合接口，避免重复建设。

### 7.1 复用接口（建议）

- `GET /api/collab/projects`：项目列表
- `GET /api/collab/projects/{id}`：项目详情
- `GET /api/collab/projects/{id}/table` 或等价表结构接口：列定义
- `GET /api/collab/projects/{id}/records`：记录分页
- `PUT /api/collab/records/{recordId}`：记录更新
- `GET /api/collab/projects/{id}/stats/*`：看板数据
- `GET /api/collab/projects/{id}/portal-links`：项目传送门

### 7.2 新增插件聚合接口（可选）

- `GET /api/plugin/collab/projects/{id}/overview`
  - 一次返回：项目信息 + 默认表结构 + 看板摘要，减少首屏请求数。
- `POST /api/plugin/auth/login`
  - 若现有登录流程不适合插件，可提供插件专用登录入口（本质仍走统一用户体系）。

### 7.3 权限与审计

- 复用现有 `canAccessProject` 权限判断；
- 所有写操作写入审计日志（操作者、项目、记录、变更前后摘要）。

## 8. VSCode 插件端技术设计

### 8.1 技术选型

- 插件框架：`vscode` Extension API
- UI：Webview + Vue3（或 React）
- 网络：`fetch/axios`
- 状态：轻量状态管理（Pinia/Zustand/自定义）

### 8.2 模块划分

- `auth`：登录、token 刷新、权限态；
- `project`：项目与表格导航；
- `table`：列表查询、编辑、冲突处理；
- `dashboard`：指标获取与图表渲染；
- `attachment`：上传、下载、预览跳转；
- `settings`：服务地址、刷新频率、保存策略。

### 8.3 配置项（插件设置）

- `autoAttend.baseUrl`：后端地址
- `autoAttend.autoRefreshSeconds`：自动刷新周期
- `autoAttend.saveMode`：`auto` / `manual`
- `autoAttend.openLinkExternally`：是否总是外部浏览器打开链接

## 9. 非功能设计

### 9.1 性能

- 首屏目标：`< 2s`（已登录、网络正常）；
- 记录分页：默认 50 条/页；
- UI 渲染采用虚拟列表（当记录数较多时）。

### 9.2 稳定性

- 请求超时与重试（幂等读请求可重试 1~2 次）；
- 后端失败需清晰报错，不吞异常；
- 插件异常写本地日志，支持一键复制诊断信息。

### 9.3 安全

- Token 使用 SecretStorage；
- 敏感配置不写入普通日志；
- 全链路 HTTPS；
- 防止 Webview XSS（严格 CSP、白名单资源）。

## 10. 迭代计划（建议）

### Phase 1（2~3 周，MVP）

- 登录鉴权
- 项目/表格导航
- 记录查询 + 基础字段编辑
- 看板只读
- 传送门只读展示

### Phase 2（2 周）

- 附件上传完善
- 评论与@协作（如后端支持）
- 冲突提示优化
- 操作审计与可观测性增强

### Phase 3（2 周）

- 传送门编辑能力
- 高级筛选与自定义视图
- 命令面板快捷操作（快速更新状态、快速新建任务）

---

## 11. 风险与应对

1. **后端接口分散、首屏请求多**
   - 应对：新增插件聚合接口，降低 RTT。
2. **多端并发编辑冲突**
   - 应对：引入版本字段校验 + 冲突弹窗。
3. **插件体验不如 Web 端完整**
   - 应对：MVP 聚焦高频核心流程，复杂能力保留跳转 Web。
4. **企业网络代理导致请求失败**
   - 应对：提供代理配置说明与连接诊断命令。

---

## 12. 验收标准（MVP）

- 开发者可在 VSCode 内完成：登录、选择项目、查看表格、修改记录并成功保存；
- 可在 VSCode 内查看项目看板核心数据；
- 权限用户可读取项目传送门并一键跳转；
- 关键异常（未登录、无权限、网络失败、冲突）均有可理解提示；
- 50 人试用后，主观“切换办公软件次数”明显下降（可通过问卷量化）。
